# 协作规则

最后更新：2026-06-30

## 强制规则

从现在开始，任何文件修改、依赖安装、服务重启、构建验证、数据库操作、配置变更，都必须先用文字说明：

- 准备做什么。
- 为什么要做。
- 会影响哪些文件或运行状态。
- 预期如何验证。

只有在得到明确授权后，才可以执行操作。

## 例外

除非用户明确允许，否则没有例外。

## 排查问题的流程

排查问题时，先只汇报计划和需要查看的信息。需要读取文件、运行命令或改动代码时，也必须先等待授权。

## 已知触发原因

本规则来自后台登录页 500 问题排查过程中的协作要求。后续必须先对齐问题、证据和修复方案，再执行修复。
